Conceitos Básicos

Netfilter: Módulo do kernel Linux que fornece funcionalidades de firewall, NAT e log de tráfego de rede.
Iptables: Ferramenta de espaço de usuário que permite criar regras de firewall e NAT.

Cadeias de Firewall

INPUT: Pacotes destinados ao próprio computador.
OUTPUT: Pacotes emitidos pelo próprio computador.
FORWARD: Pacotes que estão sendo roteados.

Ações Possíveis

DROP: Descarta o pacote.
REJECT: Descarta o pacote e envia uma mensagem ICMP de rejeição.
ACCEPT: Aceita o pacote.
LOG: Registra o pacote.

Comandos Importantes

Adicionar Regra: iptables -A <CHAIN> <REGRAS>
Ex.: iptables -A INPUT -p tcp --dport 22 -j ACCEPT
Apagar Regra: iptables -D <CHAIN> <NUMERO>
Ex.: iptables -D FORWARD 2
Substituir Regra: iptables -R <CHAIN> <NUMERO> <REGRAS>
Ex.: iptables -R FORWARD 2 -s 192.168.1.1 -j DROP
Inserir Regra: iptables -I <CHAIN> <REGRAS>
Ex.: iptables -I OUTPUT -p tcp --dport 80 -j ACCEPT
Listar Regras: iptables -L
Limpar Regras: iptables -F
Definir Política Padrão: iptables -P <CHAIN> <ACAO>
Ex.: iptables -P INPUT DROP
Criar Nova Chain: iptables -N <NOME>
Apagar Chain: iptables -X <NOME>
Renomear Chain: iptables -E <NOME_ANTIGO> <NOME_NOVO>

Parâmetros de Ação

p (protocolo): Define o protocolo (TCP, UDP, ICMP).
i (interface): Define a interface de entrada.
o (interface): Define a interface de saída.
s (origem) e d (destino): Define o endereço de origem e destino.
-sport (porta de origem) e -dport (porta de destino): Define as portas.
j (ação): Define a ação a ser tomada (DROP, ACCEPT, etc).

Criptografia

Simétrica: A mesma chave é utilizada para cifrar e decifrar a mensagem (chave compartilhada)
ALT
Encriptação é o processo de cifrar ou esconder a informação. Decriptação é o processo de converter dados encriptados de volta á sua forma original.
Substituição simples: Cada letra do texto original é substituído por uma correspondente no texto cifrado, de acordo com uma tabela baseada geralmente num deslocamento da letra original dentro do alfabeto.
Substituição homofônica: Similares as cifras de substituição simples mas um mesmo caracter do texto original pode ser mapeado para diferentes caracteres do texto cifrado.
Substituição poligrâmica: São cifras nas quais grupos de letras são tratados em conjunto ou ainda blocos de caracteres são encriptados em grupo.
Substituição polialfabética: São constituídas de múltiplas cifras de substituição simples. Uma letra pode ser associada a múltiplas chaves. Cada uma das chaves é utilizada para encriptar uma letra do texto original.
Cifragem por transposiçao: O texto original permanece o mesmo, mas a ordem dos caracteres é modificada
Esteganografia: Esteganografia é uma técnica que consiste em esconder um arquivo dentro do outro, de forma criptografada. USAR STEGHIDE

VPN

A diferença entre VPN de camada 2 (Layer 2 VPN) e VPN de camada 3 (Layer 3 VPN) reside principalmente nas camadas do modelo OSI (Open Systems Interconnection) em que operam e nas funcionalidades que oferecem. Aqui está uma explicação detalhada de cada uma e as situações em que cada uma é mais apropriada:

VPN de Camada 2 (Layer 2 VPN)

Características:

Operação: Funciona na Camada de Enlace (Layer 2) do modelo OSI.
Tecnologias Comuns: L2TP (Layer 2 Tunneling Protocol), PPTP (Point-to-Point Tunneling Protocol), VPLS (Virtual Private LAN Service).
Transparência: Transmite quadros Ethernet de uma rede para outra, permitindo que os dispositivos conectados a uma VPN de Camada 2 se comportem como se estivessem na mesma rede local (LAN).
Configuração de Rede: As configurações de rede, como endereços IP, são gerenciadas pelo cliente, ou seja, pelo dispositivo final.
Uso de Recursos: Pode ser mais complexa de configurar e manter, e pode exigir mais recursos em termos de processamento e memória.

Quando Usar:

Extensão de LAN: Quando você precisa estender uma rede local (LAN) para diferentes locais, permitindo que os dispositivos se comportem como se estivessem no mesmo segmento de rede.
Transparência de Rede: Quando é necessário que os dispositivos finais gerenciem as configurações de rede.
Compatibilidade com Aplicações: Em cenários onde aplicações específicas exigem uma conexão de Camada 2.

VPN de Camada 3 (Layer 3 VPN)

Características:

Operação: Funciona na Camada de Rede (Layer 3) do modelo OSI.
Tecnologias Comuns: MPLS VPN (Multiprotocol Label Switching), IPsec VPN.
Transparência: Transmite pacotes IP de uma rede para outra, criando túneis IP para conectar diferentes sub-redes.
Configuração de Rede: As configurações de rede, como endereços IP, são gerenciadas pelo provedor de serviços VPN ou pelo roteador VPN.
Uso de Recursos: Geralmente mais simples de configurar e gerenciar em comparação com VPN de Camada 2.

Quando Usar:

Conexão de Sub-redes: Quando você precisa conectar diferentes sub-redes ou sites que operam em diferentes faixas de endereços IP.
Simplicidade e Eficiência: Quando uma solução mais simples e eficiente é preferida, com menor complexidade de configuração e manutenção.
Escalabilidade: Em cenários onde a escalabilidade é crucial, e você precisa conectar múltiplos sites de forma eficiente.

Resumo e Escolha

VPN de Camada 2:
Use quando precisa de extensão de LAN, transparência de rede e compatibilidade com aplicações específicas.
Exemplo: Conectar escritórios em diferentes locais que precisam operar como uma única rede local.
VPN de Camada 3:
Use quando precisa de conexão de sub-redes, simplicidade de configuração e escalabilidade.
Exemplo: Conectar várias filiais de uma empresa, cada uma com sua própria sub-rede, à rede corporativa principal.
Escolha a VPN de camada 2 ou camada 3 com base nas necessidades específicas da sua rede e nas funcionalidades que deseja implementar.
